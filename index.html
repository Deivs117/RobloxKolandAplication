El Misterio de la Moneda Fantasma

El problema del nombre: Imagina que tienes un amigo que es muy literal y solo hace exactamente lo que le dices. Tú tienes una alcancía en tu cuarto que tiene un letrero que dice 'Coins'. Le das una moneda a tu amigo y le pides que vaya a guardarla en una alcancía llamada 'Money'. Como la alcancía con ese nombre exacto no existe en tu cuarto, tu amigo se confunde, se queda congelado y te lanza un error diciendo que no la puede encontrar.

El Semáforo (Debounce): Cuando pisas una moneda en la vida real, la recoges una vez y ya. Pero en la computadora, si dejas tu personaje parado sobre la moneda, ¡el juego cree que la estás tocando muchas veces por segundo! La variable db (debounce) es como un Semáforo. Cuando tocas la moneda por primera vez, el semáforo se pone en rojo. Así, aunque sigas ahí, el juego sabe que debe esperar hasta que el semáforo vuelva a verde para darte otra recompensa.

CollectScript_Bug.luau ❌ Error 
local player = game:GetService("Players")

function giveCoin(part)
    -- No hay semáforo (debounce), se cobra múltiples veces
    local player = player:GetPlayerFromCharacter(part.Parent)
    

    -- ❌ El amigo busca la alcancía equivocada ("Money")
    player.leaderstats.Money.Value = player.leaderstats.Money.Value + 1

end
CollectScript_Fix.luau ✅ Solución 
local Players = game:GetService("Players")


-- ✅ Agregamos el semáforo en verde (falso)
local db = false

function giveCoin(part)

    -- ✅ Verificamos el semáforo
    if db or part.Parent:FindFirstChild("Humanoid") == nil then
        return
    end
    db = true -- Ponemos el semáforo en rojo

    
    local player = Players:GetPlayerFromCharacter(part.Parent)
    if player then

        -- ✅ El amigo ahora busca "Coins" correctamente
        player.leaderstats.Coins.Value = player.leaderstats.Coins.Value + 1

    end
end

La Tienda Solidaria

El error del cobro: Imagina que vas a la tienda de la esquina por un helado que cuesta 3 monedas. El tendero te entrega el helado, tú te lo empiezas a comer muy feliz... ¡pero te vas corriendo sin pagarle! El tendero original era muy despistado y olvidó revisar tus bolsillos para quitarte las 3 monedas que costaba el helado. Al agregar una instrucción extra, nos aseguramos de que el tendero sea justo y recuerde cobrar exactamente lo que corresponde por el superpoder de salto.

Purchase_Script_fix.luau ✅ Solución 
local upgradeCost = 3

local function giveUpgrade(player) 
    -- [Código de búsqueda del jugador]
    if playerCoins.Value >= upgradeCost then
        
        -- Da el poder de salto
        humanoid.UseJumpPower = true
        humanoid.JumpPower = humanoid.JumpPower + 100
        

        -- ✅ Éxito: Descuenta el costo exacto del saldo
        playerCoins.Value = playerCoins.Value - 3

    end
end

El Baile de la Moneda

El bucle de tiempo: while true do es como lanzar el hechizo de un bucle de tiempo. Le dice a la moneda: '¡Haz este paso de baile para siempre!'. Pero si no le damos un microdescanso usando task.wait(), la moneda intentará girar tan rápido que el universo del juego colapsará. Ese pequeño "wait" es la respiración que toma la moneda entre cada pasito.

El Carrusel (La Orientación): Sumarle velocidad usando vectores es como ser el conductor de un carrusel. Le dices a la moneda: "No te muevas hacia arriba, no te muevas hacia los lados, solo da un pequeño giro de 3 grados sobre tu propio eje de pie" en cada respiración.

ContinuosSpin.luau ✅ Implementación 
local part = script.Parent
local speed = 3  -- grados por fotograma


while true do
    -- ✅ La respiración (1 frame ~1/60s)
    task.wait()  
    
    -- ✅ El pasito de baile (Rotación en el eje Y)
    part.Orientation += Vector3.new(0, speed, 0)
end

El Teleférico de las Dos Torres

El Director de Cine (TweenService): Mover una plataforma con código normal es muy "robótico" o torpe. El TweenService es como contratar a un Director de Cine de Animación. Tú solo le indicas dónde empieza, dónde termina y que debe tardar 7 segundos, ¡y el director se encarga de dibujar mágicamente todo el viaje súper fluido y sin tirones!

El efecto Búmeran (Parámetros mágicos): Usamos un número mágico en programación, el -1, que significa Infinito, para decirle a la plataforma que nunca deje de trabajar. Además, encendemos el interruptor del Ping-Pong poniéndolo en true. Esto hace que cuando llegue a la segunda torre, no se teletransporte feo al inicio, sino que rebote suavemente como un búmeran.

SlideMovement.luau ✅ Implementación 
local TS = game:GetService("TweenService")
local info = TweenInfo.new(
    7,                         -- ✅ Duración de 7 segundos
    Enum.EasingStyle.Cubic,
    Enum.EasingDirection.InOut,
    -1,                        -- ✅ Bucle infinito
    true                       -- ✅ Efecto Ping-Pong (búmeran)
)
local tween = TS:Create(
    script.Parent, info,
    { Position = Vector3.new(58.6, 37.598, 45) }
)
tween:Play()
